## 第5章 概率分析和随机算法

### 雇佣问题

- 雇佣问题：每天推荐一个应聘者，雇佣需要辞掉目前的助理，还要付费招聘新助理。
- 概率分析
	- 平均情况运行时间
- 随机算法

### 指示器随机变量
- 1表示发生，0表示不发生，则变量的期望表示发生概率
- 如果应聘者顺序随机，雇佣期望是ln n

### 随机算法
- 先随机排列应聘者（根据随机优先级对数组进行排序），然后取每一个更好地应聘者
- 这个算法的性能和应聘者以随机次序出现的结果相当

### 概率分析的进一步使用
- 一个屋子里有多少人，其生日落在同一天的概率是1/2？
	- 1 - k个人生日互不相同的概率是1/2
	- $1(1 - 1/n)(1-2/n)...(1-(k-1)/n)\le e^{(-i/n)}... = e^{-\sum_{i=1}^{k-1}i/n}\le 1/2$
- 两人生日相同的概率是1/n，则生日相同的一对人的数目的期望为$\sum_{i=1...k}\sum_{j=i+1...k}E(X)=C_2^k 1/n=k(k-1)/2n$
- 球和箱子
	- 有b个箱子，要投多少个球才能在给定的箱子里投中一个球？
		- 每一次投中指定箱子的概率是1/b，所以平均投b次能投中指定箱子
	- 要投多少次球能保证每个箱子里都有球？
		- 第i阶段时，i-1个箱子里有球，b-i+1个箱子是空的，命中概率为(b-i+1)/b
		- 第i阶段投球期望为b/(b-i+1)
		- 总投球次数期望为b\sum_i=1...b(1/i)=b(lnb + O(1))
- 抛硬币
	- 抛硬币n次，最长连续正面序列多长
		- 连续 $k$ 次正面的概率是 $(1/2)^k$
			- 如果你抛了 $n$ 次，你可以把它想象成大约有 $n$ 个“尝试连续正面”的机会。为了让某个长度为 $k$ 的连续序列“平均出现一次”，我们需要 $n \times (1/2)^k \approx 1$。
			- $k \approx \log_2 n$
- 在线雇佣
	- 不能再对面试者进行排序，需要马上雇佣
	- 先拒绝前k个应聘者，雇佣比前k个应聘者分数更高的第一个应聘者

## 第6章 堆排序

- 二叉堆是一个数组，是一个近似的完全二叉树
- 最大堆：亲节点比子节点都大
- 堆操作的复杂度和堆高度成正比
- MAX-HEAPIFY：维护堆的性质，若根节点小于其孩子就把根节点递归地推下去，o(lgn)
```
//调整为最大堆 ,T(n) = O(lgn)
Max_Heapify(A,idx,max) //idx：数组开始的下标，max：最大的数组下标
    left = 2*idx
    right = 2*idx
    if(left<max and A[left]>A[idx]) then
        largest = left
    else
        largest = idx
    if(right < max and A[right]>A[largest]) then
        largest = ritht
    
    if(largest != idx) then
        exchange A[largest] with A[idx]
        Max_Heapify(A,largest,max) //交换位置后，还需要调整它的子树
end
```
- BUILD-MAX-HEAP：从无序的输入数据数组中构造最大堆，从最下层非叶节点往上做MAX-HEAPIFY，O(n)
```
//建堆,运行时间的界T(n) =O(N)
BuildHeap(A)
        n = length(A)
        for  i = n/2 downto 1  do   //从非叶子节点开始，自底往上，使A变成最大堆
               Max_Heapify(A, i, n)
end
```
- HEAPSORT：原址排序，把堆顶元素弹出来，弹一次做一次MAX-HEAPIFY，O(nlgn)
```
HeapSort(A)
      BuildHeap(A)
      for i = length(A) downto 2   do 
             exchange  A[1] with A[i] //把最大堆根节点与最后一个互换
             Max_Heapify(A,1, i-1) //把交互后的排除在堆之外，重新从1到i-1,调整堆
end
```
- 优先队列：
	- MAX-HEAP-INSERT，把元素插入最大堆中，增加堆大小，设置新元素为-inf，增大它到k，O(lgn)
	- HEAP-EXTRACT-MAX，弹出并返回最大元素，把堆尾和堆顶交换，减小堆大小，然后MAX-HEAPIFY维护一下堆，O(lgn)
	- HEAP-INCREASE-KEY，将元素关键字x增加到k，增加之后把k和亲节点交换直到亲节点大于k，O(lgn)
	- HEAP-MAXIMUM：返回最大元素

## 第7章 快速排序

```
 QuickSort(A,p,r)
       if p<r
            q = Partition(A,p,r)    //确定划分位置
            QuickSort(A,p,q-1)     //子数组A[p...q-1]
            QuickSort(Q,q+1,r)     //子数组A[q+1...r]
end

//快速排序关键过程是对数组进行划分，划分过程需要选择一个主元素（pivot element）作为参照，围绕着这个主元素进划分子数组
// 双指针，i维护<=x的元素的队列，j遇到小元素了和i队列尾部交换。最后x也会被换过来
Partition(A,p,r) //p、r为数组下标
	x = A[r]   //将最后一个元素作为主元素
	i = p-1 // i指向的是比主元素小的位置，
	for  j = p  to  r-1     //从第一个元素开始到倒数第二个元素结束，比较确定主元素的位置
		if  A[j] <= x
			i = i+1 //如果比主元素小，则把i=i+1的位置上的元素和j位置发现小元素互换
			exchange A[i] <-> A[j]
	exchange A[i+1]<->A[r]   //最终确定主元的位置
	return i+1   //返回主元的位置
end
```
- 选择参考，把大的放一边，然后递归对子数组进行排序
- 如果划分嫉妒不平衡则复杂度为$\Theta(n^2)$
- 最好情况是平衡划分$\Theta(nlgn)$
- 可以随机采样主元提高随机性

## 第8章 线性时间排序

### 排序算法的下界
- 任何比较排序在最坏情况下药经过$\Omega(nlgn)$次比较
	- 因为以排序为叶节点的决策树高度大于等于lg(n!)

### 计数排序
- 对于每一个元素，确定出现次数
- 从小到大遍历每一个独特元素，把它加到结果集中出现次数次
```java
COUNTING-SORT(A, B, k)
	let C[0.. k] be a new array
	for j=1 to A.length // O(n)
		C[A[j]]=C[A[j]]+1 // cnt of elements equal to i
	for i = 1 to k // O(k)
		C[i]=C[i]+C[i-1] // cnt of elements less than or equal to i
	for j = A.length down to 1 // O(n)
		B[C[A[j]]]=A[j] // C[A[j]] serve as id, move A[j] there
		C[A[j]]=C[A[j]]-1 // decrement C[A[j]] to avoid duplicate
// that is, for A = [2,1,2,1,2], there are 2 1s and 3 2s, therefore 1 should occupy 1-2 and 2 3-5, corresponding to the item less than or equal to the item
```

### 基数排序
- n个整数排序，每个有d位数字，每个数字可以去k个值
- 从低位到高位依次按照每一位进行排序
- 排d轮，每轮$\Theta(n+k)$，共计$\Theta(d(n+k))$次
```java
RADIX-SORT(A, d)
	for i = 1 to d
		use a stable sort to sort array A on digit i
```

### 桶排序
- 假设输入数据服从均匀分
- O(n)
- 将`[0,1)`区间划分为个相同大小的桶。
	- 将n个输入数分别放到各个桶中
	- 先对每个桶中的数进行排序
	- 然后遍历每个桶，按照次序把各个桶中的元素列出来即可。
```java
n = A.length
let B[0 .. n-1] be a new array
for i = 0 to n-1
    make B[i] an empty list
for i = 1 to n
    insert A[i] into list B[⌊n * A[i]⌋] // map A[i] in [0,1) to [0...n-1]
for i = 0 to n-1
    sort list B[i] with insertion sort // 最坏O(n^2)，但每个桶中元素期望数量是1
concatenate the lists B[0], B[1], ..., B[n-1] together in order
```

## 第9章 中位数和顺序统计量
- 第i个顺序统计量(order statistic)是该集合中第i小的元素
- 如何从n个互异元素中选择第i小的统计量？

### 最小值和最大值
- 只需要最多3 floor(n/2)次比较就可以同时找到最小值和最大值
	- 将一对输入元素相互进行比较，然后把较小的与当前最小值比较，把较大的与当前最大值进行比较。
	- 这样,对每两个元素共需3次比较。
- 初始化
	- 如果是奇数,我们就将最小值和最大值的初值都设为第一个元素的值,然后成对地处理余下的元素
	- 如果是偶数, 就对前两个元素做一次比较,以决定最小值和最大值的初值

### 期望为线性时间的选择算法
基于**快速排序**中的 **划分 (Partition)** 思想，但它只递归处理包含目标元素的那个子数组，从而避免了对整个数组进行排序。
- 线性时间
```java
RANDOMIZED-SELECT (A, p, r, i) 
	if p == r // if there is only one element, it is the main element
		return A[p] 
	q = RANDOMIZED-PARTITION(A, p, r) // the current main element, when sorted, is at q
	k= q - p + 1 // calc the rank of current main element in current segment p...r
	if i == k // the main element is at the ith 
		return A[q]
	else if i < k // target smaller, search left
		return RANDOMIZED-SELECT(A, P, q-l,i)
	else return RANDOMIZED-SELECT(A, q+l, r,i-k) // search right, nitice i isdecreased to match the range
```

## 第12章 二叉搜索树
- 二叉搜索树上的基本操作所花费的时间与这棵树的高度成正比
- 对于有n个结点的一棵完全二叉树来说,这些操作的最坏运行时间为O(lgn)
	- 一棵随机构造的二叉搜索树的期望高度为O(lgn）
	- 样一棵树上的动态集合的基本操作的平均运行时间 是(lgn)
- 并不能总是保证随机地构造二叉搜索树，但可以设计二叉搜索树的变体,来 保证基本操作具有好的最坏情况性能（红黑树）

### 什么是二叉搜索树
- 对任何结点,其左子树中的关键字最大不超过x.key,其右子树中的关键字最小不 低于x.key
- 遍历顺序
	- 中序：左中右
	- 前序：中左右
	- 后序：左右中

### 查询二叉搜索树
- 下一个元素和上一个元素
	- 中序遍历中的下/上一个元素
	- 下一个元素
		- 如果有右子树，则为右子树最小
		- 否则找节点的第一个祖先，使节点在祖先的左子树中
	- 上一个元素

### 插入和删除
- 插入 O（h）
- 根据数值找到一个可以挂新节点的位置
- 根据数值把z设置为叶子节点的一个孩子
```java
TREE-INSERT(T, z)
    y = NIL // remembers the parent of x
    x = T.root
    while x != NIL // insert to an empty space
        y = x
        if z.key < x.key // if current node is smaller than old node, next look at the left subtree of the current node
            x = x.left
        else
            x = x.right
    z.p = y // set the parent of the new node as y
    if y == NIL // if T was empty
        T.root = z  // then z is the new root of the Tree
    else if z.key < y.key // if z is smaller than its parent
        y.left = z
    else
        y.right = z
```
- 删除 O（h）

- 随机构建二叉搜索树(randomly built binary search tree)为按随机次序插入这些关键字到一棵初始的空树中而生成的树

## 第13章 红黑树

### 红黑树的性质
- 一种平衡搜索树，最坏情况侠基本动态集合操作时间复杂度为O(lgn)
- 红黑树是一棵二叉搜索树,它在每个结点上增加了一个存储位来表示结点的颜色
	- K。通过对任何一条从根到叶子的简单路径上各个结点的颜色进行约束,红黑树确保没有一条路径会比其他路径长出2倍
- 树中每个结点包含5个属性: color、key、left, right 和p
- 红黑树满足
	- 每个结点或是红色的,或是黑色的。 
	- 根结点是黑色的。 
	- 每个叶结点(NIL)是黑色的。 
	- ==如果一个结点是红色的,则它的两个子结点都是黑色的==。（黑色节点的子节点可以是黑色的）
	- 对每个结点,从该结点到其所有后代叶结点的简单路径上,均包含相同数目的黑色结点。（保证==平衡==）
- 从某个结点出发(不含该结点但是含叶节点)到达一个叶结点的任意一条简单路径上的黑色结点个数称为该结点的黑高(black-height)。
	- 因为因为从该结点出发的所 有下降到其叶结点的简单路径的黑结点个数都相同，所以一个节点的黑高是固定的。
	- 红黑树的黑高为其根结点的黑高。
	- Nil的黑高为0

    else                     // 如果 x 是右孩子

    // the relationship between x and y

```python
# delete node z from T
def RB-DELETE(T, z): # z: node to delete
	y = z # a node to remove
	y-original-color = y.color
	if z.left == T.nil: # no left child, replace z with z's right subtree
		x = z.right # node for replacement
		RB-TRANSPLANT(T, z, z.right)
	elif z.right == T.nil: # no right child, replace z with z's left subtree
		x = z.left
		RB-TRANSPLANT(T, z, z.left)
	else: # both child, find a substitude for z
		y = TREE-MINIMUM(z.right) # the substitude is the minimum node in the right subtree
		y-original-color = y.color
		x = y.right # node for replacement
		if y.p == z: # if y is the right child of z
			x.p = y # if x is nil, its p pointer do not point to y originally
		else: # y is not direct right child, meaning it is someone's left child
			RB-TRANSPLANT(T, y, y.right) # removes y from its original place, y do not have a left child anyway
			y.right = z.right # make z's right subtree y's
			y.right.p = y 
		RB-TRANSPLANT(T, z, y) # replace z with y
		y.left = z.left # make z's left subtree y's
		y.left.p = y 
		y.color = z.color # change y's color to z's original color
	if y-original-color == BLACK: # why this means fixup
		RB-DELETE-FIXUP(T, x)
```

- **记录状态**：确定实际要被删除（或移动）的节点 `y`，并记录它原来的颜色（用于后续判断是否需要修复）。
- **执行物理删除/移动**：处理节点 `z` 的三种情况（无子节点、单子节点、双子节点）。
- **修复性质**：如果被删除/移动的节点是**黑色**，则树的“黑高”（Black Height）被破坏，需要调用 `FIXUP` 函数。
- z有两个孩子：找一个继承节点y替换z
	- 如果y是z的直接孩子，直接删除z。用y替代z（z的左子树变成y的左子树）。
	- 否则先从原本的位置移除y，把x（y的右节点）放在y原本的位置。然后用y替代z（z的右子树变成y的右子树，z的左子树变成y的左子树）
- 把y的颜色设置为z原本的颜色，如果z是黑色，删除z破坏了黑高，则修复x，因为y的颜色设置成了z原本的颜色，不会有问题，但是x的颜色可能有问题。

修复双重黑：如果我们删掉了一个**黑色**节点，就会导致包含该节点的路径上**黑高（Black Height）减 1**。为了暂时维持性质，我们假设接替位置的节点 `x` 身上背负了**“两重黑色”**（原本的一重 + 删掉的那一重）。

`RB-DELETE-FIXUP` 的目标就是：**把这多出来的一重黑色去掉，或者推给红色的父节点把它变黑，或者一路推到根节点把它扔掉。**

        for i = 1 to j
            if q < p[i] + r[j - i]
                q = p[i] + r[j - i]
                s[j] = i // store first seg,ent
        r[j] = q
    return r and s
