用一个大小有限的堆：大顶堆每次只能弹出最大值，我们无法移除其他数值，这样就造成大顶堆维护的不是滑动窗口里面的数值了。所以不能用大顶堆。
用一个映射记住每个现有元素和位置：不行，因为你需要知道第二大的元素。总不可能每添加一个数字进行一次排序吧

只需要维护==有可能成为窗口里最大值的元素==就可以了，同时保证队列里的元素数值是由大到小的。
	没可能成为窗口最大值的元素：比下一个进来的数小的所有数
	下一个进来的数如果比现在的数小，它还是有可能变成最大值的
设计单调队列的时候，pop，和push操作要保持如下规则：
1. pop(value)：如果窗口移除的元素value等于单调队列的出口元素，那么队列弹出元素，否则不用任何操作（这是因为push里隐含了条件如果窗口要滑出一个比top小的数，这个数已经不记录在queue里了。
2. push(value)：如果push的元素value大于入口元素的数值，那么就将队列入口的元素弹出，直到push元素的数值小于等于队列入口元素的数值为止
保持如上规则，每次窗口移动的时候，只要问que.front()就可以返回当前窗口的最大值。

```python
from collections import deque

class Solution:
    def maxSlidingWindow(self, nums: List[int], k: int) -> List[int]:
        lennums = len(nums)
        q = deque()
        for i in range(k):
            while q and q[-1] < nums[i]: # pop values smaller than next from the right
                q.pop()
            q.append(nums[i])

        results = [q[0]]
        i, j = k, 0
        while i < lennums:
            # pop nums[j]
            if nums[j] == q[0]:
                q.popleft()

            # push nums[i] 
            while q and q[-1] < nums[i]:
                q.pop()
            q.append(nums[i])

            results.append(q[0])
            i += 1
            j += 1
        
        return results
```
