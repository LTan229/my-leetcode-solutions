# 迭代
```python
class Solution:
    def deleteNode(self, root: Optional[TreeNode], key: int) -> Optional[TreeNode]:
        cur, curParent = root, None
        while cur and cur.val != key:
            curParent = cur
            cur = cur.left if cur.val > key else cur.right
        if cur is None:
            return root
        if cur.left is None and cur.right is None:
            cur = None
        elif cur.right is None:
            cur = cur.left
        elif cur.left is None:
            cur = cur.right
        else:
            successor, successorParent = cur.right, cur
            while successor.left:
                successorParent = successor
                successor = successor.left
            if successorParent.val == cur.val:
                successorParent.right = successor.right
            else:
                successorParent.left = successor.right
            successor.right = cur.right
            successor.left = cur.left
            cur = successor
        if curParent is None:
            return cur
        if curParent.left and curParent.left.val == key:
            curParent.left = cur
        else:
            curParent.right = cur
        return root
```

## GEMINI

### 总体思路

代码的执行分为三个主要步骤：

1. **查找：** 找到要删除的节点（`cur`）以及它的父节点（`curParent`）。
2. **删除和替换：** 根据`cur`有几个孩子（0个、1个或2个）来执行不同的删除逻辑。核心是找到一个合适的节点来“顶替”`cur`的位置。
3. **链接：** 将`curParent`（父节点）与“顶替”`cur`的那个新节点连接起来。

---

### 详细代码解析

#### 1. 查找节点

Python

```
cur, curParent = root, None
while cur and cur.val != key:
    curParent = cur
    cur = cur.left if cur.val > key else cur.right
if cur is None:
    return root
```

- `cur, curParent = root, None`：
    - `cur`（current）是当前遍历到的节点，从`root`（根节点）开始。
    - `curParent` 是 `cur` 的父节点，`root`没有父节点，所以初始化为 `None`。
- `while cur and cur.val != key:`：
    - 这是一个标准的二叉搜索树查找循环。
    - 只要 `cur` 不是 `None`（即没掉出树外）并且 `cur` 的值不等于我们要找的 `key`，就继续循环。
- `curParent = cur`：在 `cur` 向下移动之前，先把当前的 `cur` 存为 `curParent`。
- `cur = cur.left if cur.val > key else cur.right`：
    - 如果当前值 `cur.val` 大于 `key`，说明 `key` 应该在左子树，所以 `cur` 移动到 `cur.left`。
    - 否则（`cur.val < key`），`key` 应该在右子树，`cur` 移动到 `cur.right`。
- `if cur is None: return root`：
    - 如果循环结束时 `cur` 是 `None`，意味着我们遍历了整棵树也没找到 `key`。树未改变，直接返回原 `root`。

**至此，如果代码继续执行，`cur` 正指向要删除的节点，`curParent` 指向它的父节点。**

---

#### 2. 处理删除（核心逻辑）

这里分四种情况讨论 `cur`（要删除的节点）的孩子数量。代码用一个 `if...elif...else` 结构来处理。

**情况一：`cur` 是叶子节点（没有孩子）**

```python
if cur.left is None and cur.right is None:
    cur = None
```

- 如果 `cur` 没有左右孩子，我们直接把它设置成 `None`。后面的“链接”步骤会把 `curParent` 的对应子节点（`left` 或 `right`）设置为 `None`，从而完成删除。
    

**情况二：`cur` 只有左孩子**

```python
elif cur.right is None:
    cur = cur.left
```

- 如果 `cur` 只有左孩子（没有右孩子），我们就用它的左孩子 `cur.left` 来顶替它的位置。
    
- `cur` 这个指针现在指向了 `cur.left`。后面的“链接”步骤会把 `curParent` 指向这个新的 `cur`。
    

**情况三：`cur` 只有右孩子**

```python
elif cur.left is None:
    cur = cur.right
```

- 同理，如果 `cur` 只有右孩子（没有左孩子），我们就用它的右孩子 `cur.right` 来顶替它的位置。
    

**情况四：`cur` 有两个孩子（最复杂的情况）**

```python
else:
    # 1. 寻找后继节点 (successor)
    successor, successorParent = cur.right, cur
    while successor.left:
        successorParent = successor
        successor = successor.left
    
    # 2. 从原位置摘除 successor
    if successorParent.val == cur.val:
        successorParent.right = successor.right
    else:
        successorParent.left = successor.right
        
    # 3. successor 继承 cur 的左右子树
    successor.right = cur.right
    successor.left = cur.left
    
    # 4. 让 cur 指向 successor，准备接回父节点
    cur = successor
```

- **目标：** 为了维持二叉搜索树的性质（左 < 根 < 右），我们不能随便删除 `cur`。我们必须找到一个节点来替换它。
    - 这个替换节点通常是 `cur` 的 **“中序后继节点”**（In-order Successor），也就是 `cur` 的**右子树中最小的节点**。
- **1. 寻找后继节点 (successor)**
    - `successor, successorParent = cur.right, cur`：从 `cur` 的右孩子开始找，`cur` 暂时是 `successor` 的父节点。
    - `while successor.left:`：要找右子树中“最小”的，就是从 `cur.right` 开始，一路向左走到底。
- **2. 从原位置摘除 successor**
    - `successor` 是 `cur` 右子树中最小的节点，所以它**一定没有左孩子**（否则`while`会继续）。但它**可能有右孩子**。
    - 我们必须先把它从它原来的位置上摘下来。
    - `if successorParent.val == cur.val:`：这是一个特殊情况。如果 `cur.right` 本身就没有左孩子，那么 `successor` 就是 `cur.right`，`successorParent` 就是 `cur`。此时，`cur`（`successorParent`）的右指针应该指向 `successor` 的右孩子。
    - `else:`：这是普遍情况。`successor` 是 `successorParent` 的**左孩子**。我们让 `successorParent` 的左指针指向 `successor` 的右孩子。
- **3. successor 继承 cur 的左右子树**
    - `successor.right = cur.right`：`successor` 的新右子树，就是 `cur` 原本的右子树（注意，`successor` 已经从这个子树中被摘除了）。
    - `successor.left = cur.left`：`successor` 的新左子树，就是 `cur` 原本的左子树。
- **4. 更新 cur 指针**
    - `cur = successor`：现在 `successor` 已经完全准备好替换掉 `cur` 了。我们将 `cur` 指针（这个指针即将被链接到 `curParent`）指向 `successor`。

---

#### 3. 重新链接

在上面四种情况处理完毕后，`cur` 指针现在指向了**将要替换被删除节点**的那个节点（可能是 `None`、`cur.left`、`cur.right` 或者是 `successor`）。

```python
if curParent is None:
    return cur
```

- **特殊情况：删除的是根节点。**
- 如果 `curParent` 是 `None`，说明我们一开始要删除的就是 `root` 节点。那么树的新根就是 `cur`（它可能是 `None`，`root.left`，`root.right` 或 `successor`）。直接返回 `cur` 作为新 `root`。

```python
if curParent.left and curParent.left.val == key:
    curParent.left = cur
else:
    curParent.right = cur
return root
```

- **一般情况：链接父节点。**
- 我们检查 `curParent`：
    - 如果 `curParent` 的左孩子是我们要删除的节点（通过 `val == key` 判断），那么 `curParent.left` 就指向新的 `cur`。
    - 否则（`curParent` 的右孩子是我们要删除的节点），`curParent.right` 指向新的 `cur`。
- `return root`：
    - 因为我们处理的不是根节点（根节点的情况在 `curParent is None` 时已经返回了），`root` 指针仍然指向树的原始根。我们返回这个 `root`。

# 过程

- 找要删除的节点
	- 如果没有找到要删除的节点，就返回原来的树
	- 如果要删除的节点是叶子，就直接把他变成none
	- 如果要删除的节点只有一边孩子，则用孩子代替他
		- 能保持二叉搜索树的结构，因为一边孩子是严格比cur大或者小的，所以它也比cur的亲节点严格大或者小
	- 如果要删除的节点两边孩子都有，就找到代替他的节点
		- 找到其右子树的最左边的节点（比它大的节点中最小的）
			- 如果cur.right.left is None，则cur.right则是要找的节点，否则找它的左孩子直到没有左孩子
		- 移除这个节点
			- 如果cur.right.left = None，那么就让cur.right指向successor.right，即跳过successor就行了
			- 否则，也就是cur.right.left is not None，即successor.left is none，就删掉successor右子树最小的节点。具体而言，因为这个节点没有左孩子，所以把右孩子嫁接到succesror.parent的左子树上就可以了
		- successor取代cur
			- successor继承cur的左右子树
	- 把cur（successor）嫁接到root上
		- 如果cur就是root，那么直接返回cur
		- 否则把它安到curparent上，然后返回root
