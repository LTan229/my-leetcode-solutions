```python
nrow, ncol = map(int, input().split())
matrix = []
for _ in range(nrow):
    matrix.append(list(map(int, input().split())))

border1, border2 = [], []
reach1 = [[0] * ncol for _ in range(nrow)]
reach2 = [[0] * ncol for _ in range(nrow)]

for i in range(nrow):
    border1.append([i, 0])
    reach1[i][0] = 1
    border2.append([i, ncol - 1])
    reach2[i][ncol - 1] = 2
for j in range(ncol):
    border1.append([0, j])
    reach1[0][j] = 1
    border2.append([nrow - 1, j])
    reach2[nrow - 1][j] = 2

directions = [[1, 0], [-1, 0], [0, 1], [0, -1]]

def isvalid(i, j):
    return 0 <= i < nrow and 0 <= j < ncol

def dfs(i, j, source):
    for di, dj in directions:
        # calculate new coordinate
        newi = i + di
        newj = j + dj
        # check if valid
        if not isvalid(newi, newj):
            continue
        visited = reach1[newi][newj] != 0 if source == 1 else reach2[newi][newj] != 0
        le = matrix[i][j] <= matrix[newi][newj]
        if not visited and le:
            # mark reachable
            if source == 1:
                reach1[newi][newj] = source
            else:
                reach2[newi][newj] = source
            # further search the neighbor of newi newj
            dfs(newi, newj, source)
        
for i, j in border1:
    dfs(i, j, 1)

for i, j in border2:
    dfs(i, j, 2)

for i in range(nrow):
    for j in range(ncol):
        if reach1[i][j] == 1 and reach2[i][j] == 2:
            print(i, j)
```

### 1. 逻辑解耦：把 `visited` 数组传进去

目前的 `dfs` 函数里有很多 `if source == 1: ... else: ...` 的判断。

- **问题**：这导致 DFS 函数和全局变量耦合严重，而且每次递归内部都要做一次判断，稍微有点影响效率。
    
- **改进**：直接把需要操作的 `reach` 数组（visited 数组）当作参数传给 DFS。这样 DFS 就变成了一个纯粹的“染色”函数，它不需要知道是 source 1 还是 2，只负责把传入的数组填满。
    

### 2. 消除初始化冗余（处理四个角）

- **问题**：在你初始化 `border` 列表时：
    
    Python
    
    ```
    for i in range(nrow): ... # 包含了 (0, 0)
    for j in range(ncol): ... # 也包含了 (0, 0)
    ```
    
    矩阵的**四个角**（例如左上角 `0,0`）会被加入 `border` 两次。虽然 DFS 有 visited 检查不会死循环，但会多一次无效的函数调用开销。
    
- **改进**：不需要显式建立 `border` 列表。直接循环调用 DFS 即可。或者在循环时注意范围。
    

### 3. “先标记自己” vs “先标记邻居”

- **现状**：你现在的逻辑是“外部循环先标记起点 -> DFS 内部标记邻居”。
    
- **改进**：通常更通用的 DFS 写法是 **“进入 DFS 函数的第一件事是标记自己”**。 这样你的主循环里就不需要手动写 `reach1[i][0] = 1` 这种代码了，直接调用 `dfs(i, 0, reach1)`，函数内部会自动处理标记。代码会更简洁。

```python
nrow, ncol = map(int, input().split())
matrix = []
for _ in range(nrow):
    matrix.append(list(map(int, input().split())))

# 初始化 visited 数组
p_visited = [[False] * ncol for _ in range(nrow)] # Pacific
a_visited = [[False] * ncol for _ in range(nrow)] # Atlantic

directions = [(1, 0), (-1, 0), (0, 1), (0, -1)]

# 3. 改进后的 DFS：接收 visited_matrix 作为参数
# 逻辑：进入函数先标记自己，再找邻居。这样代码复用性最高。
def dfs(r, c, visited_matrix):
	visited_matrix[r][c] = True
	
	for dr, dc in directions:
		nr, nc = r + dr, c + dc
		
		# 边界检查
		if 0 <= nr < nrow and 0 <= nc < ncol:
			# 只有未访问过 且 能够逆流而上 (新 >= 旧) 才递归
			if not visited_matrix[nr][nc] and matrix[nr][nc] >= matrix[r][c]:
				dfs(nr, nc, visited_matrix)

# 4. 触发 DFS
# 这里的逻辑优化了：不需要手动构建 border 列表，也不需要手动预先标记
for i in range(nrow):
	dfs(i, 0, p_visited)      # 左边界 -> Pacific
	dfs(i, ncol - 1, a_visited) # 右边界 -> Atlantic

for j in range(ncol):
	dfs(0, j, p_visited)      # 上边界 -> Pacific
	dfs(nrow - 1, j, a_visited) # 下边界 -> Atlantic

# 5. 输出结果
# 这一步没有变化，逻辑很清晰
for i in range(nrow):
	for j in range(ncol):
		if p_visited[i][j] and a_visited[i][j]:
			print(i, j)
```
