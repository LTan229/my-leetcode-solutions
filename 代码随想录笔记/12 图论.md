# 并查集

- 为什么要用并查集，怎么不用个二维数据，或者set、map之类的。
	- 要添加元素的时候需要遍历所有集合
- 并查集能解决那些问题，哪些场景会用到并查集
	- 连通性问题
- 路径压缩的过程
	- 让树尽可能矮，把祖宗变成直接亲节点
- 时间复杂度分析
	- 时间复杂度在O(logn)与O(1)之间，且随着查询或者合并操作的增加，时间复杂度会越来越趋于O(1)。
	- 在第一次查询的时候，相当于是n叉树上从叶子节点到根节点的查询过程，时间复杂度是logn，但路径压缩后，后面的查询操作都是O(1)

## 功能
- 将两个元素添加到一个集合中
- 判断两个元素是否在同一个集合里

## 原理
将三个元素A，B，C （分别是数字）放在同一个集合，其实就是将三个元素连通在一起。只需要用一个一维数组来表示，即：father[A] = B，father[B] = C。如果发现根相同，则说明在一个集合

## 优化方式
路径压缩：每次查询后把当前节点的亲节点设为根节点
按秩合并：秩=高度，小树接到大树上

```c++
int n = 1005; // n根据题目中节点数量而定，一般比节点数量大一点就好
vector<int> father = vector<int> (n, 0); // C++里的一种数组结构

// 并查集初始化
void init() {
    for (int i = 0; i < n; ++i) {
        father[i] = i;
    }
}
// 并查集里寻根的过程
int find(int u) {
    return u == father[u] ? u : father[u] = find(father[u]); // 路径压缩
}

// 判断 u 和 v是否找到同一个根
bool isSame(int u, int v) {
    u = find(u);
    v = find(v);
    return u == v;
}

// 将v->u 这条边加入并查集
void join(int u, int v) {
    u = find(u); // 寻找u的根
    v = find(v); // 寻找v的根
    if (u == v) return ; // 如果发现根相同，则说明在一个集合，不用两个节点相连直接返回
    father[v] = u;
}
```

空间复杂度： O(n) ，申请一个father数组。
路径压缩后的并查集时间复杂度在O(logn)与O(1)之间，且随着查询或者合并操作的增加，时间复杂度会越来越趋于O(1)。
在第一次查询的时候，相当于是n叉树上从叶子节点到根节点的查询过程，时间复杂度是logn，但路径压缩后，后面的查询操作都是O(1)，而 join 函数 和 isSame函数 里涉及的查询操作也是一样的过程。

## Dijkstra

dijkstra算法：在有权图（权值非负数）中求从起点到其他节点的最短路径
- 权值不能为负数，否则会出现$$dist[v] + w(v, u) < dist[u]$$，dijkstra没有全局最优的能力

1. 第一步，选源点到哪个节点近且该节点未被访问过
2. 第二步，该最近节点被标记访问过
3. 第三步，更新非访问节点到源点的距离（即更新minDist数组）

**prim是求 非访问节点到最小生成树的最小距离，而 dijkstra是求 非访问节点到源点的最小距离**

## Bellman Ford

计算起始点到所有点之间的距离
- 支持负边
- 支持检测负环（边和为负值，导致最便宜路径不存在）
- 但是比较慢
$O(V \cdot E)$

```python
minDist = [float('inf')] * n
minDist[start_node] = 0

for i in range(n - 1):
	for each edge:
		minDist[dst] = min(minDist[src] + value, minDist[dst])
		
# 负环检测
has_cycle = False 
for u, v, w in edges: 
	if minDist[u] + w < minDist[v]: 
		has_cycle = True 
		break
```

根据相邻节点的信息更新每一个节点的代价n-1次
将到源点的距离初始化为0，其他所有节点的距离初始化为无穷大。对于所有边，如果通过取边缩短到目的地的距离，则距离更新为新的较低值。

如果第n轮还能发生更新，说明n个节点至少有n条边，说明成环且绕环一周代价为负

Bellman_ford算法 也是采用了动态规划的思想，即：将一个问题分解成多个决策阶段，通过状态之间的递归关系最后计算出全局最优解。

BF：松弛每一条边v-1次
SPFA：松弛过后把节点加入队列，然后只继续松弛队列里的节点

## A*

Dijkstra算法，但是不再选择最近节点，而是选择起点到当前节点距离+当前节点到终点距离最短的节点

f = g + h
- g：从起点走到当前位置已经花了多少代价
- h：估计从当前位置到终点还要花多少代价，只要满足h<=f就可以了
- f：估计从当前位置走到终点总共要花多少代价

- 选择当前f最小的节点
- 如果它是终点，则结束
- 检查它的每一个邻居
	- 如果经过当前最小节点到邻居的g值比之前记录的值小，这意味着我们发现了一条到达邻居更短的路径，同时经过当前节点和邻居的预估总代价f更小
	- 记录更好的路径，g，f
